KEY PRINCIPLES EMBEDDED IN DOCUMENTATION
==========================================

✅ Privacy-First
   Raw student behavioral data stays on edge devices (never transmitted raw)
   → See: architecture/ARCHITECTURE.md Section 4, security-compliance/SECURITY.md Section 3

✅ Event-Driven Architecture
   Agents communicate via Kafka pub/sub (loosely coupled, auditable)
   → See: architecture/ARCHITECTURE.md Section 5

✅ Human-in-the-Loop
   Teachers/admins approve high-stakes decisions (grades, parent emails)
   → See: implementation/implementation.md Section 1.4c, 1.6b

✅ Compliance-First
   FERPA, COPPA, GDPR, CCPA integrated into design (not bolt-on)
   → See: security-compliance/SECURITY.md Sections 2

✅ Transparent & Auditable
   All agent decisions logged; parents can audit data collection
   → See: architecture/ARCHITECTURE.md Section 8, security-compliance/SECURITY.md Section 5

MAINTENANCE & UPDATES
=====================

Documents are version controlled in git.

Review schedule:
  • README (project overview)       → Quarterly
  • implementation.md                → Bi-weekly during sprints
  • ARCHITECTURE.md                  → Monthly or post-design-change
  • SECURITY.md                      → Monthly or post-audit
  • Development guides               → Quarterly

When documentation needs updating:
  1. Update the markdown file
  2. Commit with clear message (e.g., "docs: update FERPA requirements")
  3. Note version bump in document header
  4. Notify team of material changes
