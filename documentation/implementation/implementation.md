## I. Project Overview

**Product Name:** Nicodemus, Teacher of Teachers
**Core Vision:** A federated, event-driven AI ecosystem that revolutionizes school operations through Privacy-First Personalization via Supabase + Modal + Inngest.

**Key Differentiators:**
- Edge-first student privacy (behavioral data stays local on Tauri app)
- Event-driven, loosely-coupled agent architecture (via Supabase Realtime + Inngest)
- Human-in-the-loop for all high-stakes decisions
- FERPA/COPPA compliance by design
- **Operational Simplicity:** No DevOps, no Kubernetes, no message queues to maintain

## IV. Implemented Features

### Phase 1a — Teacher Assistant (Curriculum & Homework)

#### Curriculum Generation
- Teachers input subject, grade level, duration, and grading system
- Modal endpoint generates a structured multi-week curriculum with lessons, learning objectives, and daily breakdowns
- Content is grade-level contextualized via `getGradeContext()` (K–12 developmental guidance)
- Auto-navigates to generated curriculum detail view on completion
- Inngest trigger is non-blocking (gracefully skipped if not configured)
- Markdown-aware parser handles `##` headers and `**bold**` content in lesson sections

#### Curriculum Detail View
- Displays: learning objectives, daily breakdown, assessment strategy, materials list
- Homework assignments listed per lesson with draft/published status badges
- Lesson variant generation (teaching style adaptation)

#### Homework System
- **AI-Generated Homework:** One-click generation from lesson context (creates `active` assignments)
- **Custom Homework Generation:** Teacher inputs a topic prompt (e.g., "Roman Government") and the system generates a standardized assignment in `draft` status for review
- **Draft → Publish Workflow:** Teachers review homework before students see it; explicit publish action moves status to `active`
- **Due Date:** Set at creation; adjustable before publishing via the draft review modal

#### Homework Question Editing
- Teachers can edit individual questions in draft assignments before publishing
- Question prompt, type, and point values are editable
- Changes saved back to the draft assignment via PATCH endpoint

### Phase 1b — PRM Agent (Parent Communications)

- Parent communication drafts generated by AI based on student metrics
- Teacher approval required before sending
- Communications log with approve/reject workflow
- Daily summary workflow via Inngest (`prm-daily-summary.ts`)

### Phase 1c — ERP Agent (Student Review HITL)

#### Teacher Review Dashboard
- Pending reviews listed with homework type and focus session type
- **Homework Review Panel:**
  - AI suggested grade displayed with 2-sentence summary (expand for more)
  - Per-answer review: student answer, teacher feedback textarea, per-answer score input
  - Calculated final grade tallied from per-answer scores in real time
  - Overall notes field for student-facing summary
  - Complete Review action triggers AI synthesis and marks submission as `graded`
- **Study Session Panel:**
  - Focus score, idle time, keystroke rate, tab switch count
  - AI summary with struggle event flags
  - Option to notify parents, add notes, approve or dismiss

#### AI Feedback Synthesis (on Complete Review)
- Annotated submission (student answers + teacher per-answer notes) sent to Modal
- AI produces synthesized overall feedback incorporating teacher observations
- Per-answer AI insights generated for answers with teacher feedback
- All stored in `per_answer_feedback` JSONB on the submission

#### Student Dashboard
- Lists pending, under-review, and graded assignments
- **Grade badge:** Shows teacher's `final_grade` (falls back to AI grade if no override)
- **Result view (after grading):**
  - Large grade display using final grade (teacher-set)
  - Teacher's overall note (amber box)
  - Grade-appropriate encouraging message
  - Per-question feedback cards: student answer, teacher feedback, AI "What to Improve" (2 sentences)
  - No duplicate overall AI analysis — feedback lives in question segments

## VI. Database Schema (Current)

### Core Tables
- `teachers` — teacher profiles, RLS by `auth.uid()`
- `students` — student profiles, linked to teachers
- `curricula` — curriculum units with subject, grade, content
- `lessons` — individual lessons within a curriculum unit
- `assignments` — homework assignments (status: `draft` | `active`)
- `submissions` — student answers, grading state, feedback columns
- `teacher_review_tasks` — HITL queue for homework + study session reviews

### Grading Columns on Submissions
- `ai_grade` — initial AI score (0–100)
- `teacher_grade` — optional per-answer sum from teacher
- `final_grade` — authoritative grade shown to student (`teacher_grade ?? ai_grade`)
- `per_answer_feedback` — JSONB array with teacher feedback + AI insight per question
- `overall_feedback` — JSONB `{ teacher_notes, ai_synthesis }`
- `submission_status` — `submitted` | `grading` | `pending_review` | `graded`

### RLS Policies
- Teachers: read/write own lessons, assignments, submissions, review tasks
- Students: read own submissions; insert own submissions; cannot read drafts
- Service role used for AI grading pipeline (bypasses RLS)

## VII. Modal Endpoints

All LLM tasks run on Modal. Endpoint base URL set via `MODAL_API_URL` env var.

| Endpoint | Purpose |
|----------|---------|
| `/generate_curriculum` | Generate multi-week curriculum with grade context |
| `/generate_lesson_variants` | Teaching style variants for a lesson |
| `/generate_homework` | Auto-generate homework from lesson content |
| `/generate_custom_homework` | Generate homework from teacher topic prompt |
| `/grade_assignment` | Pre-grade student submission with rubric |
| `/generate_parent_email` | Draft parent communication |
| `/generate_sub_lesson_plan` | Generate substitute teacher lesson plan |

All endpoints receive `grade_context` — developmental guidance for K–12 age groups.

---

## VIII. Error Handling & App Shell

- `app/error.tsx` — segment-level error boundary with retry
- `app/global-error.tsx` — root-level error boundary
- `app/not-found.tsx` — 404 page with dashboard link
- Inngest trigger is wrapped in try-catch and only fires if `INNGEST_EVENT_KEY` is set
- Modal calls in teacher review synthesizer degrade gracefully (falls back to original AI feedback)
